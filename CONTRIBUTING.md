# Contributing

感谢你参与 Claw蛐蛐。

## 工作流

1. 新建分支：`feature/...` 或 `fix/...`
2. 开发并本地验证
3. 提交 PR（描述动机、改动范围、验证方式）

## 本地校验

```bash
npm run build
```

> 提交前必须构建通过。

## Commit 规范（建议）

- `feat: ...` 新功能
- `fix: ...` 问题修复
- `chore: ...` 杂项维护
- `docs: ...` 文档更新

## 代码风格

- 保持 TypeScript 类型清晰
- 复杂逻辑写最少必要注释
- 避免引入非必要依赖

## 提交内容建议

PR 描述请包含：

- 背景/目标
- 核心改动
- 手动测试步骤
- 风险与回滚方式
